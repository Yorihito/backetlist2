# バケットリストアプリ 要件定義書

## 1. プロジェクト概要

### 1.1 アプリ概要
- **アプリ名**: バケットリスト（Bucket List）
- **目的**: 定年退職前後のユーザーが、残りの人生でやりたいことをリスト化し、実現に向けてのモチベーションを高める
- **感情ゴール**: ユーザーがアプリを使うことで「ワクワク感」を覚え、これからの人生が楽しみになる体験を提供する

### 1.2 ターゲットユーザー
- 年齢層: 60歳前後（定年退職前後）
- 特徴: サラリーマンとしての現役生活を終え、第二の人生を歩もうとしている方々
- ITリテラシー: 高くない場合もあるため、シンプルで直感的なUIが重要

## 2. 機能要件

### 2.1 必須機能（Must-Have / v1.0）

#### 認証
- Google OAuth 2.0 によるログイン・ログアウト
- ログイン済みユーザーのみアプリを利用可能

#### バケットリストアイテム管理
| 機能 | 詳細 |
|------|------|
| 追加 | テキスト入力でアイテムを追加する |
| 編集 | アイテムのタイトル・詳細・カテゴリ・重要度を変更できる |
| 削除 | アイテムを削除できる（確認ダイアログあり） |
| 一覧表示 | リスト形式で表示する |

#### カテゴリ/タグ分類
- 以下のカテゴリを初期値として提供（ユーザーによる追加も可）
  - 旅行
  - 学び
  - 経験
  - 人間関係
  - 健康・フィットネス
  - 趣味・創作
  - その他
- 1つのアイテムに複数のタグを付与可能

#### 重要度・優先度設定
- アイテム追加時に、以下の質問をユーザーに問いかける
  - 「どのくらい強くやりたいと感じますか？」（感情的な強さ）
  - 「体力や健康面で今やるべきことですか？」（時間的な制約）
- 上記の回答をもとに、システムが重要度（高/中/低）を自動設定する
- ユーザーは自動設定された重要度を手動で変更可能

### 2.2 将来対応機能（Nice-to-Have / v2.0以降）

| 機能 | 概要 |
|------|------|
| 画像・リンク添付 | アイテムに写真や参考URLを添付できる |
| ギャラリー表示 | カード形式のビジュアルな表示モード |
| 共有機能 | 家族や友人とリストを共有し、共同でアイデアを追加できる |
| リマインダー・通知 | 「今月はこのアイデアを深掘りしよう」といった通知を受け取れる |
| 投票機能 | 共有リスト内のアイテムに対して「やってみたい！」と投票できる |
| 人生目標との関連付け | バケットリストをより大きな人生の目標と紐づけて管理できる |

### 2.3 対象外（Out of Scope / v1.0）
- 進捗管理（未着手 / 計画中 / 実行済み / 完了 のステータス管理）

## 3. 非機能要件

### 3.1 対応プラットフォーム
- Webブラウザ（PCおよびスマートフォン）
- レスポンシブデザインにより、画面サイズに応じたレイアウトを提供

### 3.2 デザイン
- トーン: シックで落ち着いた色合い（ナチュラル/モダン系）
- ワクワク感を演出しつつも、視覚的に疲れないデザイン
- 文字サイズは大きめで読みやすく

### 3.3 ユーザビリティ
- ITに不慣れなユーザーでも迷わず操作できる直感的なUI
- 操作ステップは最小限にする
- エラーメッセージはわかりやすい日本語で表示する

### 3.4 セキュリティ
- 認証されたユーザーのみ自分のデータにアクセス可能
- 他ユーザーのデータへのアクセス不可（共有機能実装前）

## 4. 推奨技術スタック

| 領域 | 採用技術 | 理由 |
|------|----------|------|
| フロントエンド | Next.js (React) + TypeScript | 型安全性、SSR対応、開発効率 |
| スタイリング | Tailwind CSS | レスポンシブ対応が容易、カスタマイズ性 |
| 認証 | NextAuth.js | Google OAuth 2.0 の簡易実装 |
| バックエンド/DB | Supabase | PostgreSQL、認証、リアルタイム機能を一括提供 |
| デプロイ | Vercel | Next.js との親和性が高く、無料枠あり |

## 5. 画面構成（初期バージョン）

```
/ (ルート)
├── /login              ログイン画面（Google OAuth）
├── /dashboard          バケットリスト一覧
│   ├── アイテム追加ボタン
│   ├── カテゴリフィルター
│   ├── 重要度フィルター
│   └── アイテムリスト
└── /items/[id]/edit    アイテム編集画面
```

---

## 6. データモデル（初期バージョン）

### User
| フィールド | 型 | 説明 |
|------------|-----|------|
| id | UUID | ユーザーID |
| email | string | メールアドレス（Google OAuth） |
| name | string | 表示名 |
| created_at | timestamp | 登録日時 |

### BucketItem
| フィールド | 型 | 説明 |
|------------|-----|------|
| id | UUID | アイテムID |
| user_id | UUID | 所有ユーザーID |
| title | string | やりたいことのタイトル |
| description | text | 詳細・メモ |
| categories | string[] | カテゴリ/タグ（複数可） |
| priority | enum | 重要度（high / medium / low） |
| created_at | timestamp | 作成日時 |
| updated_at | timestamp | 更新日時 |
